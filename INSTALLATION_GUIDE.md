# คู่มือการติดตั้งระบบคลินิกทันตกรรมบ้านหมอฟัน (Professional Hybrid Setup)

คู่มือนี้สำหรับติดตั้งระบบบน **เครื่องแม่ข่าย (Server)** 1 เครื่อง และเชื่อมต่อเครื่องลูกอีก 9 เครื่องภายในวงแลนเดียวกัน ข้อมูลทั้งหมดจะถูกเก็บไว้ที่เครื่องแม่ข่าย 100% โดยเชื่อมต่อกับระบบจองคิวออนไลน์ผ่าน Supabase Cloud

---

## 🛠 สิ่งที่ต้องเตรียม (Prerequisites)
1. **Docker Desktop**: [ดาวน์โหลดที่นี่](https://www.docker.com/products/docker-desktop/) (สำหรับรันฐานข้อมูลในเครื่อง)
2. **Node.js (v18 หรือสูงกว่า)**: [ดาวน์โหลดที่นี่](https://nodejs.org/) (สำหรับรันหน้าเว็บ)
3. **โปรเจกต์โค้ด**: โฟลเดอร์ `ciki` ที่คุณมีอยู่

---

## 🚀 ขั้นตอนที่ 1: การติดตั้งบนเครื่องแม่ข่าย (Server)

### 1.1 ตรวจสอบ IP Address ของเครื่องแม่
1. กดปุ่ม `Windows + R` พิมพ์ `cmd` แล้วกด Enter
2. พิมพ์คำสั่ง `ipconfig`
3. จดจำหมายเลข **IPv4 Address** (เช่น `192.168.1.50`) **นี่คือเลขสำคัญที่จะให้เครื่องลูกใช้เชื่อมต่อ**

### 1.2 รันฐานข้อมูล Local (Supabase Docker)
1. เปิด Docker Desktop ทิ้งไว้
2. เปิด Terminal ในโฟลเดอร์โปรเจกต์ `ciki`
3. รันคำสั่ง:
   ```bash
   docker-compose -f docker-compose.local.yml up -d
   ```
   *รอจนกว่าทุกตัวจะขึ้นสถานะ `Started`*

### 1.3 ตั้งค่าไฟล์ .env
เปิดไฟล์ `.env` ในโปรเจกต์ และแก้ไขดังนี้:
```env
# ข้อมูลจองคิวออนไลน์ (Cloud)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-cloud-anon-key

# ข้อมูลภายในคลินิก (Local) - เปลี่ยนเลข IP ให้ตรงกับเครื่องแม่ของคุณ
VITE_SUPABASE_LOCAL_URL=http://192.168.1.50:3000
VITE_SUPABASE_LOCAL_ANON_KEY=your-local-key
```

### 1.4 รันระบบหน้าเว็บ
1. ใน Terminal รันคำสั่ง:
   ```bash
   npm install
   npm run dev -- --host
   ```
   *หมายเหตุ: ต้องใส่ `--host` เพื่อให้เครื่องอื่นในวงแลนมองเห็น*

---

## 💻 ขั้นตอนที่ 2: การเชื่อมต่อจากเครื่องลูก (อีก 9 เครื่อง)

ไม่ต้องลงโปรแกรมใดๆ บนเครื่องลูก เพียงแค่:
1. ต่อ Wi-Fi หรือสายแลนเดียวกับเครื่องแม่
2. เปิด Google Chrome
3. พิมพ์หมายเลข IP เครื่องแม่และ Port ที่แสดงบน Terminal (เช่น `http://192.168.1.50:5173`)
4. ใช้งานได้ทันที! ข้อมูลจะ Sync กันแบบ Real-time ทั่วทั้งคลินิก

---

## 💾 ระบบสำรองข้อมูล (Backup - สำคัญมาก!)

เนื่องจากข้อมูลเก็บไว้ในเครื่องคุณ 100% หากเครื่องพังข้อมูลจะหาย ควรทำดังนี้:
1. **สำรองข้อมูล Database**: 
   เปิด Terminal แล้วพิมพ์คำสั่ง (ทำทุกวัน):
   ```bash
   docker exec -t supabase-db pg_dumpall -c -U postgres > backup_YYYY-MM-DD.sql
   ```
2. **Copy ไฟล์**: เก็บไฟล์ `.sql` นี้ไว้ใน External Hard Drive หรือ Google Drive

---

## 💡 คำแนะนำเพิ่มเติม
- **เครื่องแม่**: ควรตั้งค่า "Never Sleep" เพื่อไม่ให้เครื่องลูกหลุดการเชื่อมต่อ
- **Router**: แนะนำให้ใช้สายแลน (LAN) สำหรับเครื่องแม่เพื่อให้ระบบเสถียรที่สุด
- **Port**: หากมี Firewall บล็อก ให้เปิด Port `3000`, `4000`, `5432` และ `5173`
